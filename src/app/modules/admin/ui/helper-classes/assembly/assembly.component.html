Assembly helper classes

Form field helpers

Following helper classes modify Angular Material's core fill form field appearance to make it more consistent and correctly aligned. You can experiment with the helper classes to see how they affect the fields.

We don't recommend using other (standard or outline) field appearances as they do not offer the best look and user experience, so to be as consistent as possible, always use the fill appearance which is the default appearance for the form fields in Assembly.

To see how we use Assembly form field helper classes correctly and efficiently, check Form Layouts examples.

Required helpers

  • Textarea (asm-mat-textarea) helper is required if you are using a textarea as your input. Since Angular Material doesn't differentiate a textarea from a normal input while using them with mat-form-field, this helper class is required for textarea inputs to look good and consistent.

Optional helpers

  • Dense (asm-mat-dense) helper provides set of adjustments to make the fields denser. Dense fields are suitable for non-form usages and places where there isn't much space to work with such as small dialogs, menus, popovers and etc.
  • Rounded (asm-mat-rounded) helper provides set of adjustments to make the fields look fully rounded.
  • No subscript (asm-mat-no-subscript) helper provides set of adjustments to remove the bottom spacing where hint and error messages placed. This helper is particularly useful to align the field easily if you don't need to show error messages or hint text. For example a search bar in the toolbar uses this helper to fit the field to the toolbar without having an extra bottom spacing.
  • Outline (asm-mat-outline) helper provides set of adjustments to make the field look like Material's Outline appearance except the top border aligned floating label.

Usage

<mat-form-field class="{{getFormFieldHelpersAsString()}}">...<mat-form-field>

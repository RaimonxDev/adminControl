Tailwind CSS

Introduction

As per the official page of Tailwind CSS, it is a highly customizable, low-level CSS framework that gives you all of the building blocks you need to build bespoke designs without any annoying opinionated styles you have to fight to override.

To simply put, Tailwind provides helper classes for almost every CSS rule available.

Assembly includes Tailwind by default and it can be used in any part of the theme. Majority of Assembly's pages and apps still use the traditional styling with ids and classes but they can be easily improved and modified using Tailwind. If you choose to use Tailwind as your main styling framework and don't want to use traditional ids and classes for styling, you can always create apps and pages and use the helper classes from Tailwind to style them.

You can also choose to use Tailwind as a helper classes utility rather than your main style framework. Basically, you would still create id/class based styles for complex styling and layouts and use only certain parts of the Tailwind to modify your styles, like quickly adding margin and padding to the elements, or showing and/or hiding them based on breakpoints to create better mobile/table layouts without writing media queries all the time.

Configuration

To use Tailwind, you don't need to do any setup as all Tailwind classes can be accessible from any HTML file you create by default.

Tailwind configuration can be found in /src/app/configs/tailwind.ts file. This file is used to configure the generated classes by Tailwind. We already configured the Tailwind using that file with the current breakpoints of the Assembly, more sizing options and couple other useful utilities.

The generated scss file is located at the /src/tailwind.scss and it's being included as one of the main style files in the building process using angular.json configuration.

Building Tailwind

If you modify the Tailwind configuration file to generate more classes or to simply edit the existing ones, you have to run the npm run tw:build command to build a new scss file with the new/modified styles. The Tailwind configuration file is not being watched by AngularCLI so any changes on that file won't trigger the Tailwind's building process automatically.
